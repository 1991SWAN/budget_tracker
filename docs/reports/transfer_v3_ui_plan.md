# Transfer V3 UI 개선 계획 (Dual Line Amount)

사용자 요청("기존 처럼 보이되 금액을 두줄로 지출 계좌 얼마 수입 계좌 얼마 하고 보여줘")을 반영하기 위한 UI 수정 계획입니다.

## 1. 개요
현재 '통합 뷰'는 출금 측(-10,000)만 보여주지만, 요청에 따라 **한 카드 내에서 출금과 입금 정보를 동시에** 두 줄로 표현합니다.

## 2. 변경 전/후 비교

| 구분 | 현재 (As-Is) | 변경 후 (To-Be) |
| :--- | :--- | :--- |
| **카드 형태** | 단일 라인 | 단일 라인 (내부 레이아웃 변경) |
| **왼쪽 (정보)** | 메모, 시간, 태그 (변동 없음) | 변동 없음 |
| **중앙 (배지)** | 카테고리, 자산(출금) | 변동 없음 (또는 필요 시 수정) |
| **오른쪽 (금액)** | **-10,000** (빨강) | **두 줄 표시**: <br> 1. [출금계좌명] **-10,000** (빨강) <br> 2. [입금계좌명] **+10,000** (초록) |

## 3. 구현 상세
- `TransactionItem` 컴포넌트 수정
- **연결된 이체(V3 Linked Transfer)**인 경우에만 우측 금액 영역을 Flex-Column으로 변경
- **Line 1 (Source)**: `asset.name` + `amount` (Red)
- **Line 2 (Destination)**: `toAsset.name` + `amount` (Green, Absolute Value)
- 모바일/데스크탑 반응형 고려

## 4. 기대 효과
- **직관성 향상**: 이체가 "어디서 어디로" 가는지 금액과 함께 한눈에 확인 가능.
- **정보보존**: 통합 뷰를 유지하면서도 숨겨진 입금 정보를 명시적으로 노출.
