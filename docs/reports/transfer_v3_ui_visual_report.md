# 링크 거래 내역 UI 상세 보고서 (Visual Perspective Only)

사용자 요청에 따라 백엔드 로직을 배제하고, **순수하게 화면에 보이는 디자인과 레이아웃 요소**만을 상세히 정리했습니다.

## 1. 개요 (Overview)
연결된 이체 내역은 기존의 단순한 한 줄 표시를 넘어, **출금과 입금 정보를 동시에 보여주는 "이중 라인(Dual Line)" 카드** 형태로 변경됩니다.

## 2. 시각적 구조 (Visual Structure)
우측 금액 표시 영역이 두 층으로 나뉘어 렌더링됩니다.

### 🔺 1층: 돈이 나가는 곳 (Source / Withdrawal)
가장 상단에 위치하며, 내 계좌의 잔고 감소를 표현합니다.
*   **레이아웃**: 좌측 은행명 + 우측 금액
*   **색상**: **Rot Rose (Red-600)** - 경고나 에러가 아닌, 명확한 '지출/감소'를 의미하는 붉은색입니다.
*   **기호**: 숫자 앞에 **마이너스(-)**가 붙습니다.
*   **텍스트**: `15px` 굵은 글씨(Bold)를 사용하여 가장 눈에 띄게 강조됩니다.

### 🔻 2층: 돈이 들어가는 곳 (Destination / Deposit)
1층 바로 아래에 위치하며, 나의 다른 계좌로 자금이 이동했음을 표현합니다.
*   **레이아웃**: 좌측 은행명 + 우측 금액
*   **색상**: **Emerald Green (Emerald-600)** - 긍정적이고 안정적인 '수입/증가'를 의미하는 초록색입니다.
*   **기호**: 숫자 앞에 **플러스(+)**가 붙습니다.
*   **텍스트**: `13px` 굵은 글씨(Bold)를 사용하되, 1층보다는 약간 작게 하여 시각적 위계를 둡니다.

## 3. 반응형 디자인 (Responsive Behavior)
화면 크기에 따라 정보의 밀도가 달라집니다.

| 화면 크기 | 은행 이름 (Asset Name) | 레이아웃 |
| :--- | :--- | :--- |
| **데스크탑** (PC, 태블릿) | **표시됨** (회색, 작은 글씨) | `은행명` + `금액` (가로 정렬) |
| **모바일** (스마트폰) | **숨김** (공간 확보) | 오직 `금액`만 두 줄로 표시됨 |

> **설계 의도**: 모바일의 좁은 화면에서는 긴 은행 이름이 금액을 가리거나 레이아웃을 깨뜨릴 수 있어 과감히 생략하고, 색상(빨강/초록)과 부호(+/-)만으로 직관적인 흐름을 파악하도록 했습니다.

## 4. 디자인 의도 요약
1.  **직관성**: 색상 대비(Red vs Green)를 통해 "돈이 나가서 여기로 들어왔구나"를 글자를 읽지 않고도 인지하게 함.
2.  **공간 효율**: 카드를 두 개로 나누지 않고 하나의 카드 안에 밀도 있게 정보를 압축.
3.  **명확성**: 단순히 `이체`라고 적힌 것보다, `Bank A -10,000` / `Bank B +10,000`으로 명시하여 오해의 여지를 없앰.

## 5. 스크린샷 분석 (Screenshot Dissection)
올려주신 이미지가 어떻게 만들어졌는지 코드를 기준으로 역설계(Reverse Engineering)하여 설명합니다.

### 🖼️ 좌측: 정보 영역
1.  **아이콘**: `↔` (양방향 화살표)
    - **로직**: `transaction.type === 'transfer'` 조건이 참일 때 이체 전용 아이콘이 렌더링됩니다.
2.  **메인 타이틀**: `허브3 → 허브2`
    - **로직**: **자산 흐름 (Asset Flow)**
    - 코드: `{asset.name} → {toAsset.name}`
    - 설명: 출금 계좌(허브3)에서 입금 계좌(허브2)로 돈이 이동함을 나타냅니다. (메모가 아님)
3.  **서브 타이틀**: `∞ Other • Transfer to Bank B`
    - **앞부분 (카테고리)**: `∞ Other`
        - 로직: `{categoryEmoji} {categoryName}`
    - **뒷부분 (추가 설명)**: `• Transfer to Bank B`
        - **로직**: 메모(Memo)에서 태그(#)와 멘션(@)을 제외한 **순수 텍스트(mainText)**
        - 설명: 사용자가 CSV나 입력창에 적은 "Transfer to Bank B"라는 메모 내용이 여기에 표시됩니다.

### 🖼️ 우측: 금액 영역 (핵심)
1.  **Line 1 (Red)**: `-10,000`
    - **로직**: `transaction.amount` (출금액)
    - **색상**: `text-rose-600` (빨강)
    - **설명**: 현재 카드(Source)에서 빠져나간 금액
2.  **Line 2 (Green)**: `+10,000`
    - **로직**: `transaction.amount` (동일 금액)
    - **색상**: `text-emerald-600` (초록)
    - **설명**: `toAssetId`와 연결된 상대방 계좌(Destination)로 들어간 금액
3.  **시간**: `09:00`
    - **로직**: `transaction.date`의 시간 부분

> **결론**:
> - **제목**은 "누구에게서 누구에게로(Hub3 -> Hub2)"를 자동으로 보여줍니다.
> - **부제목**은 "무슨 카테고리(Other)"인지와 "사용자가 적은 메모(Transfer to Bank B)"를 보여줍니다.
> - **우측**은 "빠진 돈(-)"과 "들어온 돈(+)"을 동시에 보여줍니다.
