# Import UI/UX 개선 제안서 (Import UX Improvement Plan)

## 1. 현황 분석 (Current Status)
현재 `ImportWizardModal`은 4단계 마법사(Upload -> Select -> Map -> Preview) 형태로 구성되어 있으며, 기능적으로는 완성도가 높습니다.
- **장점**: 프리셋 자동 감지, CSV 파싱, 유효성 검사, 드래그 앤 드롭 지원.
- **단점**:
    - 컬럼 매핑 시, 많은 컬럼이 있을 경우 가로 스크롤로 인해 전체 맥락 파악이 어려움.
    - "일반 모드(General)"와 "뱅킹 모드(Banking - 입/출금 분리)"의 전환이 자산 선택에 의존하여 사용자가 명시적으로 알기 어려움.
    - 매핑 미리보기 단계에서 데이터를 수정할 수 없음 (오직 제외만 가능).

## 2. 모달 UI/UX 개선 목표 (Objectives)
사용자가 더 직관적이고 빠르게 대량의 데이터를 처리할 수 있도록 인터페이스를 고도화합니다.

### A. 시각적 컬럼 매핑 (Visual Mapping Interface)
**"드롭다운 지옥(Dropdown Hell) 탈출"**
- **현재**: 각 컬럼 헤더마다 드롭다운을 클릭해서 역할을 지정해야 함.
- **개선안**:
    - **카드형 매핑**: 왼쪽에는 "필수 필드(Date, Amount, Description)" 카드가 있고, 오른쪽에는 "CSV 헤더" 리스트가 있어, **드래그 앤 드롭**으로 매칭하는 직관적 UI 도입.
    - 또는, 상단에 "목표 필드"들이 고정되어 있고, 하단 CSV 미리보기에서 해당 컬럼을 클릭하여 할당하는 "Click-to-Assign" 방식.

### B. 스마트 데이터 프리뷰 (Smart Preview & Edit)
**"가져오기 전에 고친다"**
- **인라인 편집(Inline Edit)**: 프리뷰 단계에서 잘못된 카테고리나 메모를 엑셀처럼 직접 수정할 수 있게 함.
- **카테고리 일괄 적용(Bulk Actions)**: 특정 키워드(예: "Starbucks")가 포함된 행을 선택하여 카테고리를 한 번에 "Coffee"로 지정하는 기능.
- **스마트 제안(Smart Suggestions)**: 과거 내역을 기반으로 카테고리를 자동 추천해주는 로직 시각화 (예: 🤖 아이콘 표시).

### C. 진행 상태 및 피드백 강화 (Feedback Loop)
- **명확한 모드 표시**: 현재 자산이 '신용카드(부채)'인지 '은행(자산)'인지에 따라 '입금/출금' 또는 '사용/상환'으로 용어를 동적으로 변경하여 혼동 방지.
- **오류 시각화**: 날짜 형식이 잘못된 경우, 단순히 "Invalid"라고 하기보다 "YYYY-MM-DD 형식이어야 합니다"라고 셀 바로 옆에 툴팁 제공.

## 3. 구현 로드맵 (Roadmap)
1.  **Phase 1 (Visual)**: `MappingStep`의 테이블 UI를 개선하고, 중요 컬럼(날짜, 금액)을 시각적으로 강조 (Tagging 스타일).
2.  **Phase 2 (Interaction)**: 프리뷰 단계에서 **인라인 편집** 기능 추가 (`contentEditable` 활용).
3.  **Phase 3 (Intelligence)**: 간단한 키워드 매칭 기반의 **카테고리 자동 추천** 기능 탑재.

## 4. 사용자 검토 요청
위 개선안 중 가장 우선순위가 높은 것이 무엇인지(예: 인라인 편집 vs 드래그 매핑) 말씀해 주시면 해당 부분부터 설계를 구체화하겠습니다.
