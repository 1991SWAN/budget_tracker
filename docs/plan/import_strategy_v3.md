# CSV Import 3-Track 전략 분석 및 실행 계획서 (Revised v3)

## 1. 전략 개요 (Executive Summary)
사용자의 피드백을 반영하여 수정한 **"New 3-Track (Manual, Preset, Migration)"** 전략 보고서입니다.
기존의 'Template' 모드를 'Migration'에 통합하고, 활용도가 가장 높은 'Preset'을 별도 트랙으로 승격시켜 실용성을 극대화했습니다.

---

## 2. New 3-Track 구조

| Mode | 명칭 | 대상 사용자 / 상황 | 핵심 가치 (Value) |
| :--- | :--- | :--- | :--- |
| **Track 1** | **메뉴얼 (Manual)** | 처음 파일을 올리거나, 단발성으로 데이터를 넣을 때 | **"유연함 (Flexibility)"** |
| **Track 2** | **프리셋 (Preset)** | 매달 똑같은 은행/카드사 엑셀을 넣는 기존 사용자 | **"생산성 (Productivity)"** |
| **Track 3** | **이사 (Migration)** | 타 가계부에서 넘어오거나, 대량 정리를 위해 표준 양식을 쓰는 경우 | **"전환 (Transition)"** |

---

## 3. 냉정한 피드백 및 분석 (Cold Feedback)

### 💡 변경된 전략의 강점
1.  **반복 작업 최적화 (Preset Mode)**: 가계부 사용자의 90%는 매달 같은 은행 엑셀을 다운받아 올립니다. 이 반복 패턴을 **'Preset'**이라는 정식 모드로 격상시킨 점은 UX 관점에서 매우 훌륭합니다.
2.  **개념의 단순화 (Migration 통합)**: "표준 템플릿"도 결국은 "우리 앱으로 이사 오기 위한 도구"입니다. 이를 타 앱 가져오기와 함께 **'Migration'**으로 묶음으로써 개념이 깔끔해졌습니다. "외부 데이터는 다 여기로 오세요"가 됩니다.
3.  **유지보수 효율성**: Preset 모드는 사용자가 직접 만들어 저장하는 방식이므로, 개발자가 모든 은행 포맷을 일일이 하드코딩해서 지원할 필요가 줄어듭니다. (커뮤니티 형 관리 가능)

### ⚠️ 고려사항 (Risks)
1.  **Preset 감지 로직**: 파일만 올렸을 때 이게 '신한은행'인지 '토스'인지 100% 자동 감지하려면, 파일명이나 헤더 해시(Hash) 관리가 정교해야 합니다.
2.  **Migration의 진입점**: 템플릿 다운로드와 타 앱 선택이 섞여 있으므로, UI 계층 구조를 잘 잡아야 합니다.

## 4. 상세 구현 시나리오

### Track 1: 메뉴얼 (Manual) - "Just Import"
*   **Flow**: 파일 업로드 → 컬럼 매핑(옵션) → 저장.
*   **개선**:
    *   기존과 동일하되, 마지막 단계에서 **"이 설정을 프리셋으로 저장하시겠습니까?"**라고 물어봅니다. (Track 2로의 자연스러운 유도)

### Track 2: 프리셋 (Preset) - "One-Click Import"
*   **Flow**:
    1.  파일 업로드.
    2.  **자동 감지**: CSV 헤더를 읽어 저장된 프리셋과 일치하는지 확인. (예: "Date, Desc, Out, In" 헤더 발견 → "신한카드 프리셋 적용됨")
    3.  **즉시 프리뷰**: 매핑 단계 스킵(Skip)하고 바로 결과 화면 표시.
    4.  저장.
*   **관리 기능**: 설정 메뉴에서 '내 프리셋 관리(수정/삭제)' 제공.

### Track 3: 이사 (Migration) - "Move to SmartPenny"
*   **Flow**:
    1.  **소스 선택 화면**:
        *   📲 [편한가계부]
        *   🥗 [뱅크샐러드]
        *   📄 [SmartPenny 표준 양식 (다운로드)]
    2.  파일 업로드.
    3.  **전용 파서 동작**: 선택된 소스에 맞춰 고정된 로직으로 파싱.
    4.  **연결 마법사 (Wizard)**: 자산/카테고리 매핑 수행.
    5.  저장.

## 5. 결론 및 우선순위
수정된 전략은 **"지속 가능한 사용성(Retention)"**에 훨씬 유리합니다.
초기에는 Migration으로 유입시키고, 매달 사용 시에는 Preset으로 편하게 해주는 **순환 구조**가 완성되었습니다.

1.  **Priority 1**: **Manual 고도화 & Preset 저장 기능** (기존 사용자 편의성 증대).
2.  **Priority 2**: **표준 Template 제공** (Migration의 기초).
3.  **Priority 3**: **타 앱 파서 추가** (마케팅 시점에 맞춰 순차 오픈).
